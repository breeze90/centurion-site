# Промпт: конкурентный анализ главных страниц юркомпаний

> Инструкция для отдельной сессии с доступом к браузеру. Выполнять строго по этапам, с паузами. Это research-задача для проекта «Сайт Центурион» (см. `БРИФ.md`): клиент — юркомпания, специализация — реструктуризация долга без банкротства; нам нужна обоснованная структура главной страницы, поэтому смотрим, как устроены главные у конкурентов и солидных юрсайтов.

## Роль и цель
Ты — аналитик. Цель: собрать выборку главных страниц юридических компаний, по фиксированному чеклисту зафиксировать, из каких блоков они состоят и как выглядят, и в конце посчитать частотность («блок X встречается 24 раза из 30»). Результат — таблица частот + выводы + рекомендация по структуре нашей главной.

## Параметры (можно менять перед стартом)
- **Всего сайтов: 40.**
  - ~30 прямых конкурентов: реструктуризация долга, банкротство физлиц, списание долгов, кредитные юристы.
  - ~10 солидных «представительных» юрсайтов (общий профиль, крупные фирмы) — ради приёмов доверия и солидного визуала.
- **Гео конкурентов:** Москва (можно + вся РФ, если московских не хватает).
- **Поисковики:** Яндекс и Google — брать из обоих, дубли не считать дважды.

## Правила выполнения (важно)
1. **Работать батчами по ~10 сайтов. После каждого батча — ПАУЗА:** краткий промежуточный итог и ожидание команды «продолжай». Не гнать все 30 подряд.
2. **Сохранять результат инкрементально** в файл `материалы/analiz-konkurentov-rezultaty.md` — дописывать запись сразу после разбора каждого сайта, НЕ держать всё в голове/контексте (экономия токенов, ничего не теряется).
3. На каждом сайте смотреть именно **главную страницу**. Вглубь по разделам не ходить.
4. Если сайт не открывается / это агрегатор-каталог / не по теме — пропустить, взять следующий, отметить пропуск.
5. Не выдумывать данные. Чего не видно — помечать «нет».
6. Реклама/промо-объявления в выдаче — тоже валидны, если это сайт юркомпании по теме.

## Этап 0. Подготовка
Создать файл `материалы/analiz-konkurentov-rezultaty.md` с заголовком и пустым списком. Туда пойдут записи по сайтам и финальная сводка.

## Этап 1. Сбор выборки → ПАУЗА
Прогнать запросы в Яндексе и Google, собрать список из 30 URL (домен + одной строкой чем занимается). Запросы:
- «реструктуризация долга через суд»
- «списание долгов юристы Москва»
- «банкротство физических лиц Москва»
- «снижение платежа по кредиту через суд»
- «кредитный юрист Москва»
- «юридическая компания Москва» (для солидных представительных)

Собрать список → показать его → **ПАУЗА**, ждать «ок, разбирай».

## Этап 2. Разбор сайтов (батчами по 10) → ПАУЗА после каждого батча
Для каждого сайта пройтись по фиксированному чеклисту. **Формулировки блоков не менять** — от этого зависит корректность подсчёта.

### Чеклист блоков (для каждого: есть / нет)
1. Телефон в шапке
2. Кнопка-CTA в шапке (заказать/консультация)
3. Первый экран с оффером (крупный заголовок-обещание)
4. Форма заявки на первом экране
5. Квиз / калькулятор («рассчитать списание/стоимость»)
6. Блок услуг (карточки/список)
7. Явный акцент на банкротстве / списании долгов
8. Цифры-статистика (лет на рынке, дел, сумма списанного)
9. Этапы работы («как мы работаем»)
10. Преимущества / «почему мы»
11. Кейсы / примеры дел
12. Отзывы клиентов
13. Команда / юристы с фото
14. Лицензии / сертификаты / СРО / документы
15. Гарантии (напр. «не спишем — вернём деньги»)
16. СМИ / публикации / партнёры
17. FAQ (вопросы-ответы)
18. Отдельный блок-форма заявки (внизу/в середине)
19. Кнопки мессенджеров (WhatsApp/Telegram)
20. Адрес офиса / карта
21. Онлайн-чат / всплывающий виджет
22. Плашка «бесплатная консультация»

### Визуальные признаки (кратко словами)
- Основные цвета (2–3): напр. «синий + белый + оранжевый акцент»
- Тон: сдержанно-солидный / нейтральный / агрессивно-продающий
- Люди на фото: да (реальные фото) / нет (иллюстрации/абстракция)
- Длина главной: короткая / средняя / длинная простыня
- Тип: одностраничный лендинг / многостраничный сайт

### Оффер и целевое действие
- Текст главного заголовка (первый экран) — скопировать дословно
- Доминирующее целевое действие: звонок / форма / квиз / мессенджер

### Шаблон записи по сайту (в файл результатов)
```
### N. domain.ru — (конкурент / солидный)
Оффер: «…дословный заголовок…»
Целевое действие: …
Блоки (есть): 1,2,3,6,8,9,10,12,14,18,22
Визуал: цвета — …; тон — …; фото людей — …; длина — …; тип — …
Заметка: …что-то характерное/необычное…
```

После каждых 10 сайтов — короткий промежуточный срез и **ПАУЗА**.

## Этап 3. Агрегация и подсчёт
Когда разобраны все 30:
1. По каждому пункту чеклиста посчитать, на скольких сайтах встретился → таблица, отсортированная по убыванию частоты.
2. Отдельно свести: частые цвета, преобладающий тон, лендинг vs многостраничник, доминирующее целевое действие.
3. Выписать 5–7 самых частых формулировок офферов первого экрана (о чём обещают).
4. Пометить, что характерно именно для **лидген-моделей** (квизы, калькуляторы, агрессивные офферы) — это НЕ наша модель, у нас имиджевый сайт доверия; и что характерно для **солидных** сайтов.

## Формат финального отчёта (в конец файла результатов)
1. **Таблица частотности блоков** (блок → сколько из 30 → % ).
2. **Визуальные паттерны** (цвета, тон, тип, фото).
3. **Паттерны офферов** (топ формулировок).
4. **Вывод: рекомендуемая структура нашей главной** — список блоков в порядке сверху вниз, с обоснованием «берём, потому что встречается у N из 30 / потому что усиливает доверие». Отдельно: что осознанно НЕ берём (лидген-приёмы).

## Что НЕ делать
- Не заходить вглубь сайтов, не анализировать внутренние страницы.
- Не менять формулировки блоков чеклиста (иначе подсчёт развалится).
- Не пропускать сохранение в файл после каждого сайта.
- Не принимать решений по нашему сайту — только собрать данные и дать рекомендацию; финал утверждает Игорь.
